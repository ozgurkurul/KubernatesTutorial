Çok güzel bir soru. Hemen net bir cevap vereyim: **Expo'yu bu aşamada kullanmamalıyız, kesinlikle uzun uğraştırır.**

Nedenine gelince; Expo harika bir araçtır ancak ana odak noktası **Mobil Uygulamalardır** (React Native: iOS ve Android). Web çıktısı verebilse de, arkasında mobil için tasarlanmış devasa bir kütüphane ve yapılandırma yükü getirir. 

Bizim buradaki asıl amacımız Kubernetes (K8s) süreçlerini öğrenmek olduğu için, odağımızı dağıtmayacak, sektör standardı olan, saniyeler içinde kurulan ve Dockerize etmesi çok kolay bir altyapı kullanacağız: **Vite + React**.

DevOps dünyasında "Frontend" uygulamaları genellikle **Multi-Stage (Çok Aşamalı) Docker** yapısıyla kurulur. Yani kod önce bir `Node.js` ortamında derlenir (build alınır), ardından çıkan statik dosyalar tıpkı bir önceki adımda yaptığımız gibi hafif bir `Nginx` sunucusunun içine kopyalanır.

Hadi `ozgurvm2` sunucunda bu profesyonel pipeline'ı adım adım inşa edelim.

---

### Adım 1: Proje Klasörü ve Dockerfile Hazırlığı

Sunucuda yeni bir çalışma alanı oluşturalım:

```bash
mkdir -p ~/k8s-egitim/react-app
cd ~/k8s-egitim/react-app
```

Şimdi bu klasörün içine, uygulamamızı inşa edecek olan `Dockerfile`'ı yazalım. 
*(Not: Sunucuna Node.js kurmana bile gerek yok, Docker bu işi kendi içinde halledecek!)*

```bash
nano Dockerfile
```

İçine şu "Multi-Stage" Dockerfile kodunu yapıştır:

```dockerfile
# 1. Aşama: Build (Derleme) Aşaması
FROM node:20-alpine AS builder
WORKDIR /app
# Vite ile React projesi oluştur (Normalde bunu lokalde yaparız ama K8s eğitimindeyiz)
RUN npx create-vite@latest frontend --template react
WORKDIR /app/frontend
RUN npm install
# K8s'ten geldiğimizi belli eden küçük bir dokunuş yapalım
RUN sed -i 's/Vite + React/Ozgurun K8s React Ussu/g' index.html
RUN npm run build

# 2. Aşama: Sunum Aşaması (Sadece derlenen dosyaları alır)
FROM nginx:alpine
# Builder aşamasında oluşan dist klasörünü, Nginx'in içine kopyala
COPY --from=builder /app/frontend/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### Adım 2: İmajı Oluşturma ve K3s'e Tanıtma (Önemli DevOps Dersi!)

K8s dünyasında ince bir detay vardır: Senin sunucunda yüklü olan `docker` ile K3s'in arka planda kullandığı motor (`containerd`) birbirinden bağımsız çalışır. Yani `docker build` yapsan bile K8s bu imajı göremez.

Bunu çözmek için imajı derleyip, bir dosya haline getirip K3s'in içine aktaracağız (Gerçek hayatta bu imajlar Docker Hub veya GitLab Container Registry'e push'lanır).

Sırasıyla şu komutları çalıştır:

**1. İmajı derle:**
```bash
docker build -t benim-react-imajim:v1 .
```
*(Bu işlem Node.js indireceği için 1-2 dakika sürebilir).*

**2. İmajı K3s'in içine enjekte et:**
```bash
docker save benim-react-imajim:v1 | sudo k3s ctr images import -
```

### Adım 3: K8s YAML Dosyası (Deployment + Service)

Şimdi elimizde K3s'in tanıdığı `benim-react-imajim:v1` adında dinamik bir React uygulaması var. Bunu önceki adımdaki gibi YAML ile ayağa kaldıralım.

```bash
nano react-k8s.yaml
```

İçine şu K8s manifestosunu yapıştır:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: react-deployment
  labels:
    app: dinamik-react
spec:
  replicas: 2 # Bu sefer 2 kopya (Pod) çalıştıralım
  selector:
    matchLabels:
      app: dinamik-react
  template:
    metadata:
      labels:
        app: dinamik-react
    spec:
      containers:
      - name: react-container
        image: benim-react-imajim:v1
        imagePullPolicy: Never # İnternetten arama, yereldeki imajı kullan dedik
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: react-servisi
spec:
  type: NodePort
  selector:
    app: dinamik-react
  ports:
    - port: 80
      targetPort: 80
      nodePort: 31005 # Bu sefer 31005 portundan dışarı açıyoruz
```

Kaydet ve çık. Sonra K8s'e bu durumu uygula:
```bash
kubectl apply -f react-k8s.yaml
```
