# Statik HTML Sayfası ve Nginx

Amacımız: Sadece boş bir Nginx sayfası görmek yerine, içine kendi yazdığımız `index.html` dosyasını gömen ve bunu K8s'te 3 kopya (pod) halinde çalıştıran bir yapı kurmak.

Bu süreçte K8s'in en önemli nesnelerinden biri olan **ConfigMap**'i öğreneceksin. ConfigMap, uygulamanın kodunu veya ayarlarını container imajının içine (Dockerfile ile) gömmeden, **dışarıdan** enjekte etmemizi sağlar. Bu, DevOps kültürünün temel taşlarından biridir.

---

### Adım 1: Kendi HTML Dosyamızı Hazırlama

Sunucunda (ozgurvm2) çalışacağımız temiz bir klasör oluşturalım:

```bash
mkdir -p ~/k8s-egitim/statik-web
cd ~/k8s-egitim/statik-web
```

Şimdi basit bir `index.html` dosyası yaratalım.

```bash
nano index.html
```

İçine şu HTML kodunu yapıştır:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Ozgur'un K8s Ortami</title>
    <style>
        body { font-family: Arial, sans-serif; background-color: #2c3e50; color: white; text-align: center; padding-top: 50px; }
        h1 { color: #18bc9c; }
    </style>
</head>
<body>
    <h1>Merhaba Adana'dan Kubernetes'e!</h1>
    <p>Bu sayfa K3s uzerindeki Nginx podlarindan servis edilmektedir.</p>
    <p>SSD ve 10GB RAM harika is cikariyor!</p>
</body>
</html>
```
*(Dosyayı kaydedip çık: `Ctrl+O`, `Enter`, `Ctrl+X`)*

---

### Adım 2: ConfigMap Oluşturma (Sihirli Kutu)

Şimdi bu `index.html` dosyasını K8s'in hafızasına (ConfigMap) kaydedeceğiz. Böylece K8s bu dosyayı alıp, birazdan kuracağımız Nginx pod'larının içine sanki o dosya oradaymış gibi "monte edecek" (Volume Mount).

Terminale şu komutu yazarak ConfigMap'i oluştur:

```bash
kubectl create configmap benim-html-sayfam --from-file=index.html
```

K8s'in bu dosyayı hafızasına aldığını doğrulamak için:
```bash
kubectl get configmaps
```

### Adım 3: Deployment YAML Dosyası (Nginx + HTML Entegrasyonu)

Şimdi Nginx'i 3 kopya olarak ayağa kaldıracak ve o ConfigMap'teki HTML dosyasını içine alacak olan YAML dosyasını yazalım.

```bash
nano nginx-deployment.yaml
```

İçine şu kodları yapıştır (girintilere çok dikkat et):

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ozel-nginx
  labels:
    app: statik-web
spec:
  replicas: 3
  selector:
    matchLabels:
      app: statik-web
  template:
    metadata:
      labels:
        app: statik-web
    spec:
      containers:
      - name: nginx
        image: nginx:alpine
        ports:
        - containerPort: 80
        volumeMounts:
        - name: html-volume
          mountPath: /usr/share/nginx/html # Nginx'in varsayilan HTML klasoru
      volumes:
      - name: html-volume
        configMap:
          name: benim-html-sayfam # Adim 2'de olusturdugumuz ConfigMap'in adi
```

**Ne Yaptık?**
Docker'da kullandığın `-v` (volume) parametresinin K8s versiyonunu yazdık. K8s'e dedik ki: "Bir `html-volume` yarat, içini `benim-html-sayfam` adlı ConfigMap ile doldur. Sonra bu volume'ü Nginx'in `/usr/share/nginx/html` klasörüne monte et (mount)."

Dosyayı kaydedip K8s'e uygulayalım:
```bash
kubectl apply -f nginx-deployment.yaml
```

Pod'ların 3 tanesinin de çalıştığını onayla:
```bash
kubectl get pods
```

### Adım 4: Servis YAML Dosyası (Dışarı Açma)

Son olarak, bu 3 Pod'a dışarıdan (MacBook'undan) ulaşabilmek için Santralimizi (Service) kuruyoruz.

```bash
nano nginx-service.yaml
```

İçine şu kodları yapıştır:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: ozel-nginx-servisi
spec:
  type: NodePort
  selector:
    app: statik-web
  ports:
    - port: 80
      targetPort: 80
      nodePort: 31000
```

Kaydet ve uygula:
```bash
kubectl apply -f nginx-service.yaml
```
